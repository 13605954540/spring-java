代理模式
    1:静态代理:
        优点:
            1.不影响原接口功能,扩展
            2.相对于动态代理,性能更好,动态代理使用反射
        缺点:
            1.N个接口扩展,需要N个静态代理类
    2:动态代理:
        优点:
            1.1个动态代理类,可以应付N个需要代理的接口
        缺点:
            1.基于反射,性能有影响

ps:
    1.动态代理分为jdk动态代理和cglib动态代理
    2.jdk动态代理和cglib区别
        (1).jdk只能代理实现了接口的类,cglib可以代理其他,但不能代理final的类(因为原理是生成子类继承)
        (2).jdk基于反射,cglib使用字节码技术,生成子类继承后扩展原方法.
