单例模式:
    1: 饿汉单例:
        优点:
            线程安全
        缺点:
            类加载就new出来,性能有影响

    2: 懒汉模式
        优点:
            性能好,调用getInstance才new
        缺点:
            线程不安全
            解决:
                加锁,双判断
                成员变量加volatile,避免类加载乱序造成多例的问题
    3: 静态内部类单例
    4: 枚举单例
